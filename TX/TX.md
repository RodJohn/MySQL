
# 事务

    事务可以由一条或者多条SQL组成
    事务是数据库区别于文件系统的重要特性

# 特性

    事务需要满足ACID四个特性。

A

	A(atomicity) 
	原子性
	要求事务里面的操作是一个整体，
	要么全部成功执行，要么全部失败回滚，不可以只执行其中的一部分。

C

	C(consistency) 
	一致性
	一个事务的执行不应该破坏数据库的完整性约束。

I	
	
	I(isolation) 
	隔离性
	事务之间的行为不应该互相影响。

D

	D(durability) 
	持久性
	事务一旦提交，结果就是永久的.
	即使系统崩溃，提交的数据也不应该丢失。
