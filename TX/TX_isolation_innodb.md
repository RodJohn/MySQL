
# innoDB
 
    InnoDB实现了全部的隔离级别
    
# 实现

    通过维护多个版本的数据，数据库便可以允许事务并发执行遇到互斥锁时，转而读取旧版本的数据快照。这样就能显著地提升读取的性能。我们简称这一手段为MVCC。

    隔离性由锁来实现
        
    InnoDB基于MVCC(多版本并发控制协议) 
    将读操作分成 快照读 与 当前读
    快照读
        针对于普通select语句
        通过记录的版本链和ReadView实现了全部的隔离级别
    当前读
        针对于更新语句
        通过两阶段的锁实现了全部的隔离级别
     
         
    
# 优点

    读写操作彼此并不冲突，性能更高  

    innoDB的RR的隔离级别下解决了幻读问题
