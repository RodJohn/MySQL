
# innoDB
 
    MySQL中支持事务的存储引擎有innoDB和NDB。
    InnoDB实现了全部的隔离级别
    
# 实现
        
    InnoDB基于MVCC(多版本并发控制协议) 
    将读操作分成 快照读 与 当前读
    快照读
        针对于普通select语句
        通过记录的版本链和ReadView实现了全部的隔离级别
    当前读
        针对于更新语句
        通过行锁和间隙锁实现了全部的隔离级别
    读-写操作彼此并不冲突，性能更高   
         
    
# 全功能的可重复读

    innoDB默认的隔离级别是RR，
    并且在RR的隔离级别下解决了幻读问题
